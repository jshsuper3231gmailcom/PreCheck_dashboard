# PreCheck Dashboard 로그인 보안 정책 정의서 v1.0

---

## 1. 문서 목적

이 문서는 PreCheck Dashboard에 로그인 기능을 추가하기 위한 인증/계정/세션 보안 정책을 정의한다.
대상 사용자는 증권사 내부 관리자(사내망 접근)이며, 계정 생성, 비밀번호 정책, 로그인 실패 처리,
세션 관리, 접근 제어, 화면 구성, 감사 로그 범위를 포함한다.

> 💡 1__프로그램_명세서의 "권한: 1차 버전 제외 또는 단순 사내망 접근 제어" 항목을
> 이 문서로 구체화/대체한다.

---

## 2. 인증 방식

- **Spring Security** 도입 (`spring-boot-starter-security`, `thymeleaf-extras-springsecurity6`)
- Form Login 방식, 비밀번호는 **BCrypt**로 해싱하여 저장
- DB는 운영(Altibase)/테스트(PostgreSQL) 공통이므로, 사용자/계정 관련 테이블도
  `9__로그인_DB정의서.md`의 DB 호환 원칙(VARCHAR/NUMERIC/TIMESTAMP/CHAR(1))을 따른다

---

## 3. 권한(Role) 구조

| Role | 설명 |
|---|---|
| `SUPER_ADMIN` | 계정관리 화면(계정 생성/잠금해제/비활성화/비밀번호 초기화) 접근 가능 |
| `ADMIN` | Dashboard 조회 등 일반 기능만 사용, 계정관리 화면 접근 불가 |

---

## 4. 계정 생성/관리

- 별도 가입신청 화면 없음. **SUPER_ADMIN이 계정관리 화면에서 직접 계정 생성**
  - 입력: LOGIN_ID, 사용자명(USER_NAME), ROLE, 초기 비밀번호(정책 충족 값을 관리자가 직접 입력)
  - 생성된 계정은 `STATUS=ACTIVE`, 최초 로그인 시 비밀번호 강제 변경 대상이 됨
- 계정관리 화면(SUPER_ADMIN 전용) 기능 범위
  - 계정 목록 조회 (LOGIN_ID, USER_NAME, ROLE, STATUS, LAST_LOGIN_AT, PASSWORD_CHANGED_AT)
  - 신규 계정 생성
  - 계정 잠금 해제(STATUS: LOCKED → ACTIVE)
  - 계정 비활성화/재활성화(STATUS: ACTIVE ↔ DISABLED)
  - 비밀번호 초기화 (관리자가 임시 비밀번호 지정 → 다음 로그인 시 강제 변경)
  - 계정 삭제 기능은 없음 (DISABLE로 대체, 이력 보존)

## 5. 비밀번호 정책

### 5-1. 복잡도 규칙 (계정 생성/비밀번호 변경 시 검증)

- 최소 길이 **10자 이상**
- 영문 대문자, 영문 소문자, 숫자, 특수문자 중 **3종류 이상 조합** 필수
- 동일 문자 또는 연속된 문자(숫자/키보드 순서 등) **3자 이상 반복 금지** (예: `aaa`, `123`)
- LOGIN_ID와 동일하거나 LOGIN_ID 문자열을 포함하는 비밀번호 금지

### 5-2. 비밀번호 만료 (90일)

- `PASSWORD_CHANGED_AT` 기준 **90일 경과 시 만료**
- 만료된 계정은 로그인 성공 직후 **비밀번호 변경 화면(`/password/change`)으로 강제 리다이렉트**,
  변경 전까지 다른 화면/API 접근 불가
- 만료 **7일 전부터** 로그인 시 "비밀번호 만료 D-n일" 경고 배너 표시 (변경 강제 아님, 안내만)
- **SUPER_ADMIN 시드 계정은 만료 정책 예외** (`PASSWORD_EXPIRE_YN='N'`)

### 5-3. 비밀번호 재사용 금지 (전/전전 비밀번호)

- 비밀번호 변경 시 **최근 2개(직전 + 그 이전) 비밀번호**와 동일한 값으로 재설정 불가
- `TB_ADMIN_USER_PWD_HISTORY`에 변경 시점마다 BCrypt 해시 저장, 신규 비밀번호를
  최근 2건의 해시와 `matches()` 비교하여 동일하면 거부

---

## 6. 로그인 실패 / 계정 잠금 정책

- **5회 연속 로그인 실패 시 계정 잠금** (`STATUS=LOCKED`, `LOCKED_AT` 기록)
- 잠금된 계정은 **잠금 시각으로부터 5분 경과 시 자동 해제** (`STATUS=ACTIVE`, `LOGIN_FAIL_COUNT=0`)
- SUPER_ADMIN은 계정관리 화면에서 **5분 대기 없이 즉시 수동 해제** 가능
- 로그인 성공 시 `LOGIN_FAIL_COUNT=0`으로 초기화, `LAST_LOGIN_AT` 갱신
- 잠긴 계정으로 로그인 시도 시 "계정이 잠겼습니다. 관리자에게 문의하세요" 메시지만 표시
  (남은 시도 횟수, 잠금 해제 시각 등 구체 정보는 노출하지 않음)

---

## 7. 세션 정책

- 세션 타임아웃(무활동 기준) **9시간**
- **동일 계정 동시(다중) 로그인 전부 허용** (`maximumSessions` 제한 없음)
- 로그아웃 시 세션 무효화 + 쿠키 삭제
- 내부망 HTTP 고정 운영을 전제로 쿠키 설정
  - `Secure=false`
  - `HttpOnly=true`
  - `SameSite=Lax`

## 8. 접근 제어 (URL 매핑)

| URL 패턴 | 접근 조건 |
|---|---|
| `/login`, `/css/**`, `/js/**`, `/plugins/**`, `/images/**`, `/fonts/**` | 인증 불필요 |
| `/dashboard/**` (페이지 + API) | 인증 필요 (ADMIN, SUPER_ADMIN) |
| `/admin/users/**` (계정관리) | `SUPER_ADMIN` 권한 필요 |
| `/password/change` | 인증 필요 (모든 인증 사용자) |

- 인증되지 않은 상태로 보호된 페이지 접근 시 `/login`으로 리다이렉트
- 인증되지 않은 상태로 `/dashboard/api/**` 호출 시 **401 JSON 응답**

## 9. 화면 구성

- `/login`
  - AdminLTE 로그인 카드 스타일 (기존 admin-lte 리소스 활용)
  - LOGIN_ID / PASSWORD 입력, 인증 실패 시 에러 메시지 표시
- `/password/change`
  - 현재 비밀번호 + 신규 비밀번호(+확인) 입력
  - 최초 로그인/비밀번호 만료로 강제 진입한 경우 취소 불가
- 기존 Dashboard 헤더(`precheck-header-meta` 영역)
  - 로그인 사용자명 표시
  - "비밀번호 변경" 링크
  - "로그아웃" 버튼
- `/admin/users` (계정관리, SUPER_ADMIN 전용)
  - 계정 목록, 생성, 잠금해제/비활성화 토글, 비밀번호 초기화

## 10. 감사 로그

다음 이벤트를 `TB_ADMIN_USER_LOG`에 기록한다.

| ACTION_TYPE | 기록 시점 |
|---|---|
| `LOGIN_SUCCESS` | 로그인 성공 |
| `LOGIN_FAIL` | 로그인 실패 (계정 존재 여부와 무관하게 시도한 LOGIN_ID 기록) |
| `PASSWORD_CHANGE` | 비밀번호 변경 (값 자체는 기록하지 않음) |
| `USER_CREATE` | 계정 생성 |
| `USER_LOCK` / `USER_UNLOCK` | 계정 잠금 / 잠금 해제(자동·수동 모두) |
| `USER_DISABLE` / `USER_ENABLE` | 계정 비활성화 / 재활성화 |
| `PASSWORD_RESET` | 관리자에 의한 비밀번호 초기화 |

각 이력은 대상 계정, 수행자(ACTOR, 본인 행위는 동일), 클라이언트 IP, 시각을 포함한다.
